# v0.7.2 - Repeated Final ㅅ Rule

## Problem

`0+3+8+8+8` produced `았ㅆ`.

## Expected

`0+3+8+8+8` must produce `았ㅅ`.

## Rule

When a repeated consonant has already converted a final consonant once,
the next identical key press starts a new consonant instead of replacing the final again.

Examples:

```text
0+3+8       = 앗
0+3+8+8     = 았
0+3+8+8+8   = 았ㅅ

1+33+9+8+8   = 겠
1+33+9+8+8+8 = 겠ㅅ
```
