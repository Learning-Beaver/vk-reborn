# v0.7.1 Fix: 겠 입력

## Problem
`1+33+9+8+8` should become `겠`, but the v0.7 engine treated repeated consonants after a final consonant as the next syllable in all cases.

## Rule
When the current buffer is `C + V + C` and the same key is repeated:

- If the repeated output can be a valid final consonant and differs from the previous final, replace the final consonant.
  - `게 + ㅅ + ㅅ => 겠`
  - `가 + ㄱ + ㄱ => 갂`
- Otherwise, append as the next syllable initial.
  - `안 + ㄴ => 안ㄴ`

## Test Cases
- `1+33+9+8+8 => 겠`
- `0+3+2+2+33+한+0 => 안녕`
- `0+99+0+0+9 => 응이`
