# v0.9.5 Tap / LongPress Separation

## Fix

빠른 반복 입력 중 이전 탭의 롱프레스 예약 콜백이 다음 탭에 붙어 실행되는 문제를 수정했다.

### 재현 케이스

```text
0 + 3 + 2 + 2 + 3 + 3 + 한 + 0
```

기대값:

```text
안녕
```

기존 오류:

```text
안2ㅕㅇ
```

## Cause

v0.9.2 Turbo에서 일반 입력을 ACTION_DOWN 즉시 처리하도록 바꾸면서 롱프레스 예약 Runnable이 ACTION_UP에서 제거되지 않았다.

빠르게 같은 키를 다시 누르면 이전 Runnable이 살아 있다가 `downKey == key` 조건을 만족해 롱프레스로 오인될 수 있었다.

## Changes

- `longPressRunnable` 보관
- ACTION_UP / ACTION_CANCEL / ACTION_MOVE에서 롱프레스 예약 제거
- `touchSequence`로 이전 터치 이벤트의 지연 콜백 무효화
- 롱프레스 임계값을 300ms로 조정

## Regression Test

```text
0+3+2+2+33+한+0 -> 안녕
0+99+0+0+9 -> 응이
빠른 2222 입력 시 숫자 2가 끼어들지 않아야 함
롱프레스 ㄴㅁ -> 2 입력
DEL 길게 -> 연속 삭제
```
