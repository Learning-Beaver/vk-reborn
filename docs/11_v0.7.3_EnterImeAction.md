# v0.7.3 - ENTER / IME Action Fix

## Problem
Chrome or other search fields inserted whitespace/newline when ENTER was pressed.

## Fix
ENTER now checks `EditorInfo.imeOptions` and calls `performEditorAction()` for:

- SEARCH
- GO
- SEND
- DONE
- NEXT
- PREVIOUS

If no action is specified, it falls back to sending `KEYCODE_ENTER`, preserving normal newline behavior in memo/text editor apps.

## Test Cases

- Chrome search box + ENTER -> search executes
- Browser address bar + ENTER -> navigation/search executes
- Memo app + ENTER -> newline
- Messenger input + ENTER -> app-defined behavior
