# v0.9.1 Performance Patch

## 목표
빠른 타이핑 시 입력이 씹히는 현상을 줄이고 키보드 반응속도를 개선한다.

## 반영 내용
- 키 영역(Rect) 계산을 `onDraw()`마다 수행하지 않고 `onSizeChanged()`/모드 변경 시 캐싱
- 전체 `invalidate()` 호출을 줄이고 눌린 키 영역만 부분 갱신
- Shift 상태 변경 시 전체 키보드가 아니라 Shift 키 영역 위주로 갱신
- `Handler.removeCallbacksAndMessages(null)` 제거
  - 빠른 연속 입력 시 다음 키의 예약 이벤트까지 제거될 수 있는 위험 방지
- 롱프레스 대기 시간을 420ms에서 360ms로 조정
- composing text 갱신 시 `beginBatchEdit()/endBatchEdit()` 적용

## 테스트 기준
- `안녕하세요안녕하세요` 빠르게 입력
- `qwertyuiop` 빠르게 입력
- `1234567890` 빠르게 입력
- `DEL` 길게 삭제
- 영문 Shift / Caps Lock 상태 표시 정상 여부

## 남은 성능 개선 후보
- ACTION_DOWN 즉시 입력 방식 검토
- 키 팝업 렌더링 최적화
- Canvas bitmap cache 적용
- Debug FPS/Profile overlay 추가
